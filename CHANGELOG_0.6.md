# Changelog 0.6.0

## Браузерная автоматизация

- **Единый переключатель** «Браузерная автоматизация» в настройках (Options, Panel, Popup): один тумблер включает/выключает оба инструмента `page_click` и `page_fill`. При выключении ассистент не видит и не вызывает их.
- Настройка хранится в `chrome.storage.sync` как `browserAutomationEnabled` (по умолчанию `false`).
- В UI чекбокс заменён на **тумблер** (переключатель) с подписью; стили в panel.css и popup.css.
- Локализация: ключи `settings.browserAutomation`, `settings.browserAutomationDescription` (en/ru).

## Клик и заполнение полей на странице

- **page_click**: клик по кнопке, ссылке или элементу по тексту/селектору (content script + background).
- **page_fill**: ввод текста в поле (input/textarea) по метке, placeholder, name, id или селектору; отправка `input`/`change` для совместимости с React и др.
- **Поиск поля «поиск»/«запрос»**: синонимы (search, поиск, запрос, query, q, find, искать, …) и учёт `type="search"`, `role="searchbox"`; приоритет видимого поля.
- **Дедупликация**: при повторных одинаковых вызовах `page_click`/`page_fill` в одном раунде выполняется один раз, повторные ответы только в `messages`, в UI показывается один блок «Вызов инструмента».
- **XML-парсер**: если модель возвращает вызовы в виде `<function=page_fill>` … `</function>` в тексте (без native tool_calls), они парсятся и выполняются; дубликаты по name+args отфильтровываются.

## Семантика инструментов и контекст

- Описания инструментов переписаны по **намерению** (activate on page → page_click; put text into field → page_fill), без жёсткой привязки к формулировкам.
- В системный промпт добавлен короткий блок **[INTENT]**: сопоставление намерения и инструмента (click → page_click, type/fill → page_fill, search web → web_search), указание «no exact wording needed».

## Поиск в интернете

- Новый встроенный инструмент **web_search**: открывает в новой вкладке результаты поиска (без бэкенда и без API).
- Движки: **DuckDuckGo** (по умолчанию), **Google**, **Yandex**. Параметры: `query` (обязательный), `engine` (необязательный).
- Добавлено разрешение **tabs** в manifest для `chrome.tabs.create`.
- В [INTENT] добавлено: search the web / look up → web_search.

## Прочее (из предыдущих сессий)

- Множественные LLM-конфиги (чипы в Options, выбор по имени в чате), миграция со старых настроек, API-ключи в `chrome.storage.local`.
- Автосохранение настроек, удаление кнопки «Сохранить» и кнопки «Проверить MCP»; при ошибке MCP в статусе «Error loading».
- Тема (светлая/тёмная/системная) — тумблер в шапке; автоопределение хоста и моделей по кнопке Auto.
